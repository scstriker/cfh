# 以宋老师的版本为主



我要开发一个软件，将各位专家撰写的标准草案“合并”为合并稿，这里的合并有两层含义

不是简单的交集或者并集，而是在深刻理解内容后，寻找重合的部分再有机整合

严格禁止外部搜索！ 



具体要求，细则如下：

1. 部署在aistudio上的纯前端页面，调用gemini api key；
2. 导出为符合国家标准模板要求的docx文档；
3. 合并稿中以醒目的样式显示内容的来源，比如这句话说来自于哪个文档；
4. 生成合并稿的时候，可以选择以一个或多个指定的文档为合并稿的主要来源，例如本项目中以国家标准-原子级制造术语-宋凤麒.docx 为主；

不着急一次性给可行性、实施方案你和技术栈，先吃透几个草案再想想合并的两层含义，尤其是第一层，分步骤来
